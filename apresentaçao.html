Objetivo do Projeto

Criar uma plataforma de monitoramento responsiva capaz de gerenciar volumes de dados de transporte público em tempo real. O foco principal é realizar transições de estados fluidas: partindo de uma visão macroscópica da cidade inteira até o foco microscópico em uma linha específica, mapeando seus pontos de parada e calculando distâncias precisas para o passageiro.

O Esqueleto: index.html

O arquivo atua como o orquestrador de containers da aplicação. Ele cria a estrutura semântica inicial e reserva os IDs estratégicos onde o mapa do Leaflet e os contêineres de mídia (como o vídeo de carregamento) serão injetados dinamicamente, preparando a árvore do DOM antes de qualquer processamento lógico.

Integração: index.html

Nesta camada, estruturamos os componentes visuais de controle superiores através do Bootstrap 5.3. O arquivo gerencia a inclusão centralizada das dependências externas por CDNs, garantindo que o motor do Leaflet, os scripts de geoprocessamento da Proj4js e os parsers do PapaParse estejam disponíveis na memória global da aplicação.

CSS — style.css

Classes definidas

  • .mapa — controla o tamanho e a posição do mapa na tela
  • .onibus-video — aplica a animação de passagem ao vídeo do ônibus
  • .caber — posiciona o contêiner do vídeo na página
  • @keyframes passar — define o movimento do ônibus da direita para a esquerda em loop infinito

Propriedades do .mapa

  • width: clamp(0.1vw, 60vw, 70vw) — largura responsiva
  • height: clamp(0.1vh, 45vh, 70vh) — altura responsiva
  • margin: auto — centraliza horizontalmente
  • margin-top: clamp(10px, 4vw, 6vw) — espaçamento do topo responsivo
  • border-radius: 20px — cantos arredondados
  • display: flex com align-items e justify-content: center — centraliza o conteúdo interno

Função clamp()

  • Sintaxe: clamp(mínimo, ideal, máximo)
  • Faz o valor crescer e diminuir conforme o tamanho da tela
  • Mantém o layout responsivo sem precisar de media queries
  • Usada em largura, altura e margens do mapa

Animação @keyframes passar

  • from: translateX(80vw) — o ônibus começa fora da tela, à direita
  • to: translateX(-100%) — o ônibus sai completamente pela esquerda
  • Configuração: 10s linear infinite — duração de 10 segundos, ritmo constante, sem pausa

Bootstrap 5.3

  • Biblioteca CSS carregada via CDN no HTML, sem instalação
  • Fornece componentes e classes prontas para estilização
  • Classes usadas no projeto:
    • navbar-dark bg-dark — barra de navegação com fundo escuro e texto claro
    • btn btn-warning — botão amarelo "Filtrar" com estilo pronto
    • btn-outline-light — botão "Limpar" transparente com borda branca
    • d-flex gap-2 — organiza o input e os botões lado a lado com espaçamento

JS 1: Estrutura Global

Neste bloco, o sistema define os Alicerces de Dados. Ele aloca os arrays globais que funcionarão como um cache em memória para armazenar as paradas da cidade, além de inicializar as variáveis que rastreiam os marcadores ativos e as coordenadas do usuário, evitando releituras custosas de arquivos locais.

JS 2: O Desafio UTM

Atua como o Tradutor Geográfico do projeto. Como a base de dados oficial da prefeitura registra as coordenadas em metros (sistema projetado UTM Zone 23S), este bloco emprega os algoritmos matemáticos da biblioteca Proj4js para convertê-las em graus decimais esféricos (Latitude e Longitude), o formato padrão exigido pelo Leaflet.

JS 3: Processamento CSV

Utiliza a biblioteca PapaParse para realizar a leitura pesada de arquivos locais de forma assíncrona. O script varre o documento de paradas e emprega Expressões Regulares (RegEx) para isolar o texto da geometria textual, limpando ruídos e montando uma coleção estruturada de dados geográficos prontos para consulta.

JS 4: Palco Visual

Instancia o construtor do Leaflet.js centralizado em Belo Horizonte. A arquitetura aqui baseia-se em Layer Groups (grupos de camadas), tratando elementos visuais como folhas transparentes sobrepostas. Isso possibilita limpar os ônibus antigos e plotar novas rotas instantaneamente, sem a necessidade de recarregar o mapa de fundo.

JS 5: A Ponte de Dados

Configura o canal seguro de comunicação com a PBH. Para contornar as restrições nativas de segurança dos navegadores que bloqueiam requisições diretas entre domínios diferentes (erros de CORS), a aplicação encapsula a chamada de rede através de um CORS Proxy público, mantendo estável o tráfego em tempo real.

JS 6: Ciclo de Monitoramento

É o Motor Gráfico de atualização da frota. Ele consome o JSON do servidor externo de forma assíncrona e gerencia o redesenho dos veículos. Para preservar a performance em dispositivos móveis e evitar lentidão causada pelo excesso de imagens complexas, utiliza marcadores circulares vetoriais leves na exibição global.

JS 7: Inteligência de Filtro

Gerencia as Transições de Estado baseadas nas entradas do usuário. Quando o passageiro insere o código de uma linha, este bloco interrompe o estado de exibição geral, limpa o mapa de veículos irrelevantes e altera as variáveis de contexto para direcionar a aplicação ao trajeto específico.

JS 8: Visão da Linha

Responsável pelo Enquadramento Automático da tela. O script faz uma varredura sobre as paradas filtradas do ônibus para encontrar as coordenadas geográficas limites do trajeto e invoca a função fitBounds do Leaflet, ajustando o foco da câmera para exibir todo o itinerário na tela instantaneamente.

JS 9: Proximidade Inteligente

Aplica a lógica de Cálculo de Distância Espacial. Utilizando a API de Geolocalização do navegador para posicionar o usuário (marcador verde), o algoritmo realiza uma iteração contínua sobre as paradas da rota selecionada. O ponto com o menor valor de distância é localizado e destacado com alertas interativos.

JS 10: Orquestração Final

Representa o Relógio Síncrono da aplicação. Através de temporizadores nativos de repetição (setInterval), ele estabelece que a cada 20 segundos o ecossistema realize automaticamente um novo ciclo completo de busca à API da prefeitura, garantindo dados sempre novos sem intervenção do usuário.
